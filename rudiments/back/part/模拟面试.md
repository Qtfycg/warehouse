### 4.1.Java基础

```text
1.Java 内存结构中，堆（Heap）和栈（Stack）的区别是什么？它们分别存放哪些数据？
    Java栈是线程私有的，主要存放局部变量和对象引用，方法结束栈帧自动弹出。堆是线程共享的，用于存放所有对象实例和数组，由垃圾回收管理器管理生命周期。栈的分配速度快，没有并发问题；堆结构复杂，需要垃圾回收，性能相对慢。
2.详细说一下HashMap的底层结构？它是如何解决哈希冲突的？扩容机制是怎样的？
    底层是数组、链表、红黑树的组合结构。数组用来定位bucket。链表解决哈希冲突，使用的是链地址法，即多个元素落在同一个数组位置就用链表或红黑树挂起来。当前元素数量超过阈值时触发扩容，会把数组大小扩成原来的2倍，并对原有节点做再哈希。
3.==和equals的区别是什么？为什么重写equals时通常也要重写hashCode？
    ==比较基本类型的值或引用地址而equals比较对象内容，如果只重写equals而不重写hashCode，就可能出现equals相等但hashCode不同的情况，导致相同对象无法匹配，出现数据重复或查找失败。
4.请你说一下synchronized的底层实现原理是什么？锁的升级过程是怎样的？
    synchronized底层基于对象头和Monitor实现，在无竞争情况下使用偏向锁，同一线程反复加锁几乎零成本；有轻微竞争时升级为轻量级锁，通过CAS和自旋解决；当竞争激烈时自旋失败，再升级为重量级锁，线程会阻塞挂起。锁升级是不可逆的，这样可以减少锁状态切换的成本，提高 synchronized 的性能
5.线程池的核心参数有哪些？线程池是如何避免频繁创建线程带来的开销的？
    线程池的核心参数有七个，分别是核心线程数、最大线程数、任务队列、时间单位、空闲线程存活时间、线程工厂，以及拒绝策略，通过线程复用机制避免频繁创建线程，一个线程可以持续处理多个任务，并通过核心线程、队列、最大线程三阶段调度策略，控制线程创建频率，空闲的非核心线程会在存活时间后销毁，保证资源不被占用。
6.请说一下JVM常见的垃圾回收算法有哪些？为什么要分代垃圾回收器？
    常见GC算法包括：标记-清除、标记-整理、复制算法以及引用计数，之所以要分代，是因为对象的生命周期差异很大，分代GC能避免每次扫描整个堆，从而显著提升整体 GC 性能。
7.请你说说 final、finally、finalize 的区别？
    final用于修饰类、方法、变量，表示不可变。finally是异常处理中一定会执行的代码块，常用于资源释放。finalize 是对象被垃圾回收器回收前的回调方法，但不可靠，已废弃，不推荐使用。
8.请解释一下 Java 的值传递机制。为什么说 Java 只有值传递而没有引用传递？
    Java只有值传递。对基本类型，传入的是值的副本；对对象，传入的是‘引用地址值的副本’，所以能修改对象内容，但不能让实参引用指向其他对象。如果是引用传递，那么改变形参引用会影响实参，因此 Java 不存在引用传递。
9.请说说 Java 中 String、StringBuilder、StringBuffer 的区别和使用场景？
    String是不可变的，每次修改都会创建新对象，适用于内容不变的。StringBuilder是非线程安全的，内容可变，适用于单线程大量拼接；StringBuffer是线程安全的，内容可变，适用于多线程安全拼接。
10.请你说说 Java 的自动装箱与拆箱机制是什么？有哪些容易出错的地方？
    自动装箱是把基本类型转换为包装类，本质是调用valueOf()；自动拆箱是把包装类转为基本类型，本质是调用xxxValue()，包装类参与运算时会自动拆箱，因此实际开发中建议高频运算使用基本类型。
11.Java的异常体系结构是怎样的？Checked Exception 和 Unchecked Exception 的区别是什么？
    Java异常体系以Throwable为根，分为Exception和Error，Exception又分为受检异常与非受检异常。受检异常必须在编译期处理，非受检异常不强制处理，多来自程序逻辑错误，需要开发者修复而不是捕获。实际开发中，推荐使用统一异常处理统一捕获并包装异常，让系统返回标准化响应格式。
12.请你讲一下Java的继承与多态机制？运行时多态是如何实现的？
    Java支持继承和多态，子类可以扩展和重写父类。多态分为重载（编译时）和重写（运行时）。运行时多态的实现依赖动态分派，JVM 会在类加载阶段为每个类建立虚方法表，对象调用方法时根据实际对象的虚方法表找到重写后的实现，因此父类引用可以调用子类方法。虚方法表是Java运行时多态的核心。
13.请解释重载（Overload）与重写（Override）的区别。
    重载发生在同一个类，方法名相同但参数列表不同，用于增强方法适用性，是编译期多态；而重写发生在父子类之间，方法名、参数列表完全一致，用于让子类有自己的实现，是运行期多态。
14.请你解释接口（interface）与抽象类（abstract class）的区别？
    接口强调规范和能力，可以多实现，不包含对象状态，只能有常量和抽象行为。而抽象类强调代码复用，可以定义成员变量和普通方法，只能单继承，适合作为父类提供部分默认实现。
15.请你简述 Java 类加载机制的过程包括哪几个阶段？每个阶段主要做什么？
    Java 类加载机制包括加载、验证、准备、解析、初始化、使用、卸载等七个阶段。加载是把类文件读入内存并生成类对象、验证确保字节码合法、准备阶段为静态字段分配内存并设置默认值、解析将常量池中的符号引用替换为直接引用、初始化执行静态变量赋值和静态代码块，是类加载的最后一步，整个流程由JVM和类加载器共同完成。
16.请解释 transient、volatile、synchronized 三个关键字的区别与作用？
    transient用于序列化，表示字段不参与序列化、volatile是轻量级同步，只保证可见性和禁止指令重排，但不保证原子性，适合状态标志、synchronized 是重量级同步，保证原子性、可见性和有序性，通过对象头和 Monitor 实现，适合临界区互斥访问。
17.请你说一下volatile和synchronized的底层实现区别，以及在什么场景下适合使用 volatile？
    volatile通过内存屏障保证可见性和禁止指令重排，但不保证原子性，是无锁方案，不会阻塞线程。而synchronized底层依靠对象头的Mark Word和Monitor实现互斥，支持偏向锁、轻量级锁到重量级锁的升级，能提供原子性、可见性和有序性，是完整的同步手段。volatile适合状态标志、配合CAS使用的场景；但涉及原子操作或复合逻辑必须用synchronized。
18.请简述 Java 泛型的原理？什么是类型擦除？它的优缺点是什么？
    Java泛型的本质是编译器在编译期做类型检查，运行时通过类型擦除把泛型转换为原始类型。类型擦除意味着所有泛型信息在运行时都不存在，只保留Object或上界类型。其优点是保持向下兼容并避免类膨胀，但缺点是在运行时无法获取泛型类型、不能创建泛型数组，也不能使用基本类型。
19.请解释一下 Java 内存模型（JMM）的主要作用是什么？它解决了哪些问题？
    Java内存模型用于规定多线程读写共享变量的规则，是并发编程的基础，主要解决三类问题：可见性、原子性、有序性。
20.请解释一下ThreadLocal的原理，它是如何实现线程隔离的？
    ThreadLocal通过在每个线程内部维护一个ThreadLocalMap来实现线程隔离。ThreadLocal只是key，真正的数据存在每个线程自己的ThreadLocalMap中，因此每个线程都有独立的数据副本，不会互相干扰。如果ThreadLocal被回收但线程长期不结束，会造成value泄漏，所以用完要手动remove。其典型用途包括保存用户上下文、格式化工具等线程独有的数据。
```

```text
1.简述一下MySQL的整体架构，包括SQL执行的主要流程？
    MySQL架构为插件化架构，由服务层和引擎层构成。服务层包括连接器、解析器、优化器和执行器，负责SQL的解析、优化和执行逻辑，存储引擎层则负责数据的存储与索引管理，二者通过通过Handler接口进行交互，InnoDB是最常用的引擎，提供事务、行锁、MVCC 等能力。SQL的执行流程是：先通过连接器建立连接 → 解析 SQL → 经过优化器生成执行计划 → 执行器按照执行计划逐步执行 → 最后由存储引擎根据索引读取数据并返回结果。
2.请你说说MySQL索引的底层结构是什么？为什么使用B+树而不是B树或哈希？
    MySQL索引底层使用B+树，相比B树，B+树非叶子节点不存储数据，单页能容纳更多key，使得树更矮，磁盘IO次数更少。且叶子节点通过链表连接，非常适合范围查询、排序和顺序读。Hash虽然等值查找快，但不支持范围查询、排序、前缀匹配，因此无法作为数据库的通用索引结构。
3.解释一下InnoDB的聚簇索引与非聚簇索引的区别？以及什么是回表？
    主键索引是聚簇索引，它的叶子节点存储整行数据，一张表只能有一个聚簇索引。非聚簇索引的叶子节点不存整行，只存索引列和主键值。因此通过非聚簇索引查询时，还需要根据主键再次去聚簇索引中查完整记录，这个过程叫回表。如果查询字段都能在二级索引中找到，就称为覆盖索引，可以避免回表，提高性能。
4.解释一下MySQL的事务特性和四种隔离级别，以及它们各自可能出现的问题？
    MySQL事务具备原子性、一致性、隔离性、持久性，有四种隔离级别，分别是是读未提交、读已提交、可重复读、串行化。读未提交会有脏读、不可重复读和幻读；读已提交避免脏读但仍有不可重复读和幻读；MySQL默认的可重复读通过MVCC解决不可重复读，通过间隙锁解决幻读；串行化隔离级别最高但性能差。
5.详细说明InnoDB的锁类型有哪些？分别解决什么问题？
    InnoDB有表级锁、行级锁、插入意向锁、自增锁、元数据锁等锁类型，且都是基于索引实现，如果不走索引会退化为表锁。表级锁即意向锁，而行级锁则包含记录锁、间隙锁、临键锁，记录锁用于锁定具体记录，间隙锁锁定区间避免插入，临键锁是记录锁+间隙锁，避免幻读。插入意向锁允许多个插入并发进行，提高插入效率，自增锁保证自增值正确，元数据锁保证表结构修改时不会与读写冲突。
6.在MySQL中，哪些情况会导致索引失效？
    MySQL索引常因函数/表达式、隐式类型转换、模糊查询、不等查询、OR 非索引列、联合索引未命中最左前缀、对字段进行计算导致失效。
```
